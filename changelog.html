—— Project Origins ——
v1.0 – v2.7
Early 2025 · Built with Google Gemini
Origin
Development History

Meeting Tools was originally conceived and built in collaboration with Google Gemini. Development proceeded through v2.7, establishing the core architecture and feature set. As the codebase grew in complexity, Gemini encountered limitations handling the scope of changes required, and development transitioned to Claude (Anthropic) starting at v2.8. Claude has handled all development from that point forward.

The following is Gemini's own functional specification of the app at the point of handoff:

Identity & Navigation
  • Dynamic header branding (55px bold) updating contextually to Timing Tools or Counter Tools
  • Club Identity — tapping header allows persistent club name configuration
  • iOS-style full-width segmented tab container
Roster, Timer & Ah-Counter
  • Live predictive filtering, auto-fill on name tap, localStorage persistence with alphabetical sort
  • Timer presets (Table Topics, Evaluation, Speech, Ice Breaker, Custom) with full-screen color transitions
  • Tally cards for AH, UM, ER, SO, LIKE, YOU KNOW with tap-to-increment and minus button
Reporting & Technical Standards
  • Consolidated meeting log with one-touch clipboard copy
  • Safety-checked Clear All, OpenAI key field in Settings
  • Centered 900px responsive container for iPad/tablet use
  • Single-file architecture — all CSS, JS, and HTML in one file to eliminate external dependency failures

Cloud Meeting Report
Discussed Feb 25 session
High
Problem

Timer and Ah-Counter are run by different people on different devices. There's no way for the VP-PR to pull a combined meeting report without asking everyone to manually share their local reports.

Solution
  • Add a "Cloud Report" view to the existing Report screen
  • Queries today's Firestore sessions across all devices, filtered by date automatically
  • Assembles into the same Name | Role | Result format as the local report
  • Write active WOD to Firestore club document on Select — appears in cloud report
  • Use case: VP-PR opens app after the meeting at home, taps Cloud Report, sees the full picture for the newsletter
—— Medium Priority ——
Manual Filler Word Scoring
Discussed Feb 25 session
Medium ~2 hrs
Problem

Manual filler tracking produces raw counts only. Scoring (filler rate %, Excellent / Good / Needs Work) is currently only available via the Whisper transcription path — which requires an OpenAI key.

Solution
  • Hidden background timer starts when the counter screen loads at START
  • Timer stops when user taps SAVE RESULTS
  • Estimate word count at 130 wpm (standard prepared speech pace)
  • Run identical scoring math as the transcription path
  • Display same result card — total fillers, estimated word count, filler rate %, color-coded score
  • Note shown: "Word count estimated based on elapsed time"
  • Session record gains duration, estimatedWords, fillerRate, scoringMethod: "manual"
Design Decision — Pauses

Pauses and interruptions are intentionally ignored. If the clock runs long the speaker gets a slightly better score that session, but over multiple sessions the Progress average reflects their true pattern. The noise washes out.

Role Manager
Discussed Feb 25 session
Medium Prereq for PACE ~3–4 hrs
Problem

Club roles are hardcoded in renderSetupGrid(). Clubs cannot add, rename, or customize roles to match their chapter's practices, and there's no way to assign timer thresholds or PACE point values per role.

Three Systems Stay Independent
  • Roles — what a member did at a meeting (drives Timer grid, Counter grid, PACE points)
  • Filler words — what a member said during their speech (Ah-Counter tally cards — no changes to this flow)
  • PACE — accumulates points from roles over time across meetings

The Ah-Counter workflow is completely unchanged: pick speaker → pick role → tap tally cards → save. The role grid just becomes dynamic instead of hardcoded.

PACE Race Tracker
Discussed Feb 25 session
Medium Requires Role Manager ~4–6 hrs
What PACE Is

Participating and Achieving Competence and Excellence — a point-based incentive system where members earn points by taking on meeting roles, targeting 100 points as the goal. Point values come directly from the Role Manager and are fully customizable per club.

Default Point Values (configurable in Role Manager)

In-App Features
  • New PACE section — meeting date selector (defaults to today), assign roles to roster members
  • Points calculated automatically from Role Manager values
  • Leaderboard — cumulative points sorted descending, progress bar toward 100-point goal
  • Tap any member to see full role history with dates and points per entry
Embeddable Chart for Club Websites
  • Single <script> tag embed with club sync code as a parameter
  • Fetches pace records from Firestore (public read scoped to pace subcollection only)
  • Renders a bar chart via Chart.js — members on X axis, cumulative points on Y, goal line at 100
  • Self-contained, no iframe, renders cleanly on any background

Google Auth + Admin Panel
Discussed Feb 25 session
Medium ~1 day
Problem

No admin capability exists — dead clubs accumulate, there's no usage visibility, and there's no way to manage elevated access without giving everyone the same sync code.

Authentication

Google Sign-In via Firebase Auth — 20-minute Firebase Console change, no new service. Club members keep the existing sync code system unchanged. Google Auth required only for admin panel access.

Role-Based Access Control
RolePermissions
ownerFull access — add/remove admins, assign roles, delete any club, view all stats, purge dead clubs
adminView stats, archive/delete inactive clubs — cannot manage other admins
viewerRead-only — stats and club list, no modifications
Admin Panel Features
  • Dashboard — total active clubs, total members tracked, total sessions, sessions this month
  • Club list — name, sync code (masked), member count, session count, last activity date
  • Inactive club management — configurable threshold (default 6 months), soft-delete first, hard-delete after 90-day grace period
  • Admin user management (owner only) — add Google email, assign role, remove access
Delivery

Separate admin.html — never linked from the app, accessed by direct URL only. Owner bootstraps by adding their own email to the admins collection in Firebase Console on first setup.

Offline Session Write Resilience
Discussed Feb 25 session
Medium
Problem & Solution

writeSession() is fire-and-forget — if a device is offline at Commit & Exit the session silently never reaches Firestore. Firebase SDK has built-in offline persistence that queues writes and replays automatically when connectivity resumes. Enabling it handles this transparently with no UX changes.

Low risk for typical WiFi meeting environment. Bump priority if members report missing sessions.
